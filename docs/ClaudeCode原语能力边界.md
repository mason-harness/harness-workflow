# Claude Code 四原语能力边界（参考）

> **来源说明**：本文件基于 Claude Code 官方能力说明做归纳整理，仅作为四类原语能力边界的参考说明。

## 1. 四原语速查

| 原语 | 主要触发方式 | 适合承载什么 | 不适合承载什么 |
| --- | --- | --- | --- |
| Hook | 生命周期事件 | 必然发生、可机械判断、需要拦截或注入上下文的动作 | 长流程编排、跨会话状态、需要人工交互的逻辑 |
| Skill | 手动 `/skill-name` 或任务匹配自动触发 | 某阶段才需要的多步流程、参考资料、模板化做法 | 必须硬阻断的门禁、会话级恒定规则 |
| Rule | 会话启动时自动加载 | 每次会话都应生效的稳定指令与边界 | 执行动作、硬拦截、一次性任务流程 |
| Subagent | Claude 委托或 workflow/skill 派发 | 一次性探索、监测、校验、并行独立任务 | 长期持久化规则、跨会话状态、越权操作 |

## 2. Hook

### 2.1 触发方式

固定生命周期事件，例如 `PreToolUse`、`PostToolUse`、`SessionStart`、`UserPromptSubmit`、`FileChanged`。

### 2.2 能做什么

- 在工具执行前拦截并阻断（`exit 2`）或放行
- 向上下文注入额外信息（`additionalContext`）
- 以 shell 命令、HTTP POST、MCP 工具调用、LLM 评估等形式运行
- 配合 matcher 过滤特定工具、会话来源或错误类型

### 2.3 不能做什么

- 无控制终端，不能访问 `/dev/tty`
- 无法跨会话持久化状态
- 不适合超过 30 秒的执行，尤其 `UserPromptSubmit` 会阻塞模型处理
- 输出不能超过 10,000 字符
- 复杂 Bash 条件判断解析可能不稳定，失败时可能退化为放行

## 3. Skill

### 3.1 触发方式

用户手动通过 `/skill-name` 调用，或 Claude 在识别到相关任务时自动触发。

### 3.2 能做什么

- 封装多步骤流程或参考资料，并按需懒加载
- 通过 frontmatter 控制调用方式，例如自动、仅手动、优先手动
- 在主 agent 上下文或独立 subagent 中运行
- 携带模板、配置或参考文件
- 通过 `once: true` 控制自动调用时每会话仅执行一次

### 3.3 不能做什么

- 不能跨会话持久化状态
- 不能强制拦截或阻断动作，这属于 Hook 的职责
- 不能扩展 Claude 本身没有的工具权限

## 4. Rule（`CLAUDE.md` / `.claude/rules/`）

### 4.1 触发方式

会话启动时自动加载进上下文。

### 4.2 能做什么

- 给出每次会话都应生效的持久指令
- 通过 frontmatter `paths:` 按文件路径范围加载
- 拆分为多个主题文件按需加载
- 使用 `@path` 语法引入其他文件
- 通过 managed settings 定义组织级策略，并随 git 仓库跨 worktree 或分支生效

### 4.3 不能做什么

- 不能执行代码或产生副作用，它本质上只是 Claude 会读取的文本
- 不能硬性强制执行，仍属于软约束
- 不能自动持久化学习内容，那属于 auto memory 的职责

## 5. Subagent（Agent）

### 5.1 触发方式

Claude 自主判断委托，或由 Skill、workflow 显式派发。

### 5.2 能做什么

- 隔离执行，拥有独立上下文窗口和自定义系统提示
- 限制可用工具集，形成最小权限执行面
- 路由到更快或更便宜的模型处理特定工作
- 并行派发多个独立任务
- 保护主上下文不被大量探索、检索或监测内容撑爆

### 5.3 不能做什么

- 子 agent 之间不能直接通信，只能各自回传结果
- 不能天然跨会话持久化，除非把结果写入文件
- 不能绕开主 agent 未授权的权限边界去执行破坏性操作

## 6. 对照速查

| 维度 | Hook | Skill | Rule | Subagent |
| --- | --- | --- | --- | --- |
| 触发 | 生命周期事件 | 手动或自动识别 | 会话启动 | Claude 自主委托 |
| 执行方式 | 同步阻塞 | 主上下文顺序执行或派发 | 仅注入文本 | 隔离执行 |
| 强制性 | 硬，可拦截 | 软，偏流程编排 | 软，偏长期指令 | 中，偏上下文与权限隔离 |
| 上下文代价 | 按次调用 | 懒加载 | 启动时加载 | 主上下文可显著减负 |
| 持久性 | 无 | 无 | 文本长期存在 | 无，结果回传后结束 |
